 Deliverable B — Auditor Narrative

This design is APPI-minded because regulated application data is stored only in Tokyo (`ap-northeast-1`), while São Paulo (`sa-east-1`) is used for compute and user-facing application processing only. The database query evidence shows that `arcanum-rds01` exists in Tokyo and that São Paulo has no RDS instance, so storage residency is preserved. Public access enters through CloudFront, is inspected by WAF, and does not reach the ALB directly from the internet. Cross-region access is not open-ended; it is constrained to a documented TGW peering corridor between `tgw-0f43bb3559a4397f3` and `tgw-07cf5b59d76c3556c`. CloudTrail records who changed the security group rules, logging destinations, and audit trail configuration, creating a traceable management-event history. CloudFront logs and WAF logs provide access and security evidence, while versioned and encrypted S3 buckets provide retention posture. CloudTrail digest files and `validate-logs` confirm that audit logs can be integrity-checked, which is critical in a regulated environment. For those reasons, the architecture supports controlled global access while keeping authoritative storage in Japan and preserving an auditor-ready chain of evidence.
